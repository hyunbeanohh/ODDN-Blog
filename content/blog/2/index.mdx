이 문서는 **목차 프로토타입 검증용**입니다.

- 데스크톱: 우측 고정 목차 표시
- 모바일: 접고 펼치는 목차 표시
- 제목 클릭: 해당 섹션으로 이동
- 스크롤 중: 현재 섹션 하이라이트

## Executive Summary

이번 테스트의 목적은 MDX 본문에서 `h2/h3`를 추출해 목차를 자동 생성하는 UX를 검증하는 것입니다.

### 핵심 목표

사용자가 긴 문서에서도 현재 위치를 잃지 않고 빠르게 원하는 섹션으로 이동할 수 있어야 합니다.

### 기대 효과

문서 탐색 시간이 줄고, 이탈률과 탐색 피로가 감소합니다.

## 정보 구조 설계

목차에는 `h2/h3`만 포함하고, `h3`는 한 단계 들여써 위계를 표현합니다.

### 포함 규칙

목차 항목은 의미 단위가 명확한 제목만 노출합니다.

### 제외 규칙

너무 세부적인 제목(`h4` 이하)은 기본 설정에서 제외합니다.

## 상호작용 설계

클릭 시 URL 해시를 갱신하고 해당 제목으로 이동합니다.

### 활성 섹션 표시

현재 화면 상단에 가장 가까운 섹션을 활성 상태로 표시합니다.

### 스크롤 동작

부드러운 스크롤을 적용하되 `prefers-reduced-motion` 환경에서는 즉시 이동합니다.

## 접근성 체크

목차 컨테이너는 `nav`와 `aria-label`을 사용하고 현재 항목은 `aria-current`를 부여합니다.

### 키보드 탐색

탭 이동으로 모든 목차 항목에 접근할 수 있어야 합니다.

### 시각적 대비

활성/비활성 상태의 대비를 충분히 두어 구분 가능하게 합니다.

## 중복 제목 테스트

중복 제목 slug 생성이 충돌 없이 동작해야 합니다.

### 같은 소제목

중복 소제목 첫 번째입니다.

### 같은 소제목

중복 소제목 두 번째입니다.

## 성능 확인 포인트

IntersectionObserver 기반으로 스크롤 이벤트 남용을 피합니다.

### 관찰 대상 수

문서가 길어져도 관찰 대상은 제목 요소로 제한합니다.

### 리렌더 최소화

목차 데이터는 초기 스캔 후 재사용하고 불필요한 상태 변경을 줄입니다.

## 결론

현재 프로토타입은 탐색성과 구조 가독성 개선에 초점을 둔 첫 버전입니다.
